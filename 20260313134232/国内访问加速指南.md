# 🇨🇳 国内访问加速指南

由于 Netlify 服务器在国外，国内访问可能会比较慢。以下是几种解决方案：

## 方案一：Cloudflare CDN 加速（推荐，免费）

### 步骤：

1. **购买域名**（如果还没有）
   - 推荐在阿里云、腾讯云、Namecheap 购买
   - 便宜的 .com/.cn 域名约 50-70 元/年

2. **在 Netlify 绑定自定义域名**
   - 登录 Netlify → 选择你的站点
   - 进入 "Domain settings" → "Add custom domain"
   - 输入你购买的域名（如：yourdomain.com）
   - 按照提示添加 DNS 记录

3. **在 Cloudflare 配置 CDN**
   - 注册 Cloudflare 账号（免费）：https://dash.cloudflare.com
   - 添加你的域名
   - 修改域名的 DNS 服务器为 Cloudflare 提供的
   - 开启 CDN 加速（橙色云朵图标）
   - 等待 DNS 生效（通常几分钟到几小时）

### 优点：
- 免费使用
- 全球 CDN 节点加速
- 提供 DDoS 防护
- 支持 HTTPS

## 方案二：部署到国内云服务平台

### 推荐平台：

#### 1. 腾讯云 CloudBase（云开发）
```bash
# 安装 CLI
npm install -g @cloudbase/cli

# 登录
tcb login

# 部署
tcb deploy
```

**优点：**
- 免费额度充足
- 国内访问速度快
- 自带数据库和存储

**缺点：**
- 需要实名认证
- 部分功能需要备案

#### 2. 阿里云函数计算
- 支持静态网站托管
- 国内访问速度快
- 按量付费，费用低

#### 3. Vercel（有国内节点）
- 和 Netlify 类似
- 部分用户反馈国内访问比 Netlify 快

---

## 方案三：使用镜像站点

### 1. Gitee Pages（码云）
- 国内访问速度快
- 免费使用
- 支持静态网站托管

**限制：**
- 需要实名认证
- 需要备案才能绑定自定义域名

### 2. GitHub Pages + 国内镜像
- 使用 ghproxy 等镜像加速
- 适合开源项目

## 方案四：优化现有 Netlify 站点

### 已做的优化：
1. ✅ 添加了静态资源缓存（JS/CSS/图片缓存1年）
2. ✅ 减少了 API 请求数据量
3. ✅ 添加了前端数据缓存

### 额外建议：

1. **使用图片压缩**
   - 上传前压缩图片
   - 使用 WebP 格式
   - 减少图片尺寸

2. **懒加载图片**
   - 只加载可视区域的图片
   - 滚动时再加载其他图片

3. **使用 Service Worker 缓存**
   - 离线访问支持
   - 减少网络请求

## 快速推荐方案

### 如果你不想花钱：
1. 继续使用 Netlify
2. 开启 Cloudflare CDN（免费版）
3. 使用免费的二级域名（如：yourname.pages.dev）

### 如果你愿意花少量钱（约 70 元/年）：
1. 购买一个域名（阿里云/腾讯云）
2. 绑定到 Netlify
3. 使用 Cloudflare CDN 加速
4. 国内访问速度会有明显提升

### 如果你需要最佳国内体验：
1. 使用腾讯云 CloudBase
2. 购买国内域名并备案
3. 部署到国内服务器
4. 访问速度最快，但需要备案

---

## 备案说明

如果你使用国内服务器或国内 CDN，需要进行 ICP 备案：

- **个人备案**：约 7-20 天，免费
- **企业备案**：约 7-20 天，免费
- 需要：身份证、手机号、域名证书等

**免备案方案：**
- 使用香港/海外服务器
- 使用 Cloudflare 等海外 CDN
- 使用 Netlify/Vercel 等海外平台
